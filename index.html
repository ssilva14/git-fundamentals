Master Git & GitHub for Effective Project Management

A Step-by-Step Guide

This comprehensive guide equips you with essential skills for mastering Git and GitHub, key tools for effective project management and version control. Start by creating a GitHub account and setting up your first repository to organize your projects.

Next, install Git and Visual Studio Code to streamline file management. Explore the powerful features of remote repositories to push updates and pull changes, keeping your work current. Additionally, learn to create pull requests and engage in collaborative code reviews, enhancing your teamwork capabilities.

By the end of this guide, you'll possess valuable skills that are highly sought after in today’s tech industry. Let’s elevate your development skills and get started!

1. Create GitHub Account

Before you install Git and Visual Studio Code, start by creating a GitHub account:

  • Go to github.com and click "Sign up."
  • Enter your email, password, and a username.

2. Create a New Repository

This section will guide you through creating your first repository on your GitHub account:

  • Sign In: Go to github.com and log in with your username and password.
  • Go to Your Dashboard: After signing in, you’ll see your GitHub home page.
  • Start a New Repository: Click the “New” button or the “Create a new repository” button on the right. You can also click the "+" icon in the top-right corner.
  • Set Up Your Repository:
    • Repository Name: Type a name for your project.
    • Visibility: Choose Public (anyone can see it) or Private (only you and people you invite can see it).
  • Create the Repository: Click the “Create repository” button to finish.
What is a Repository?

A repository is like a digital folder on GitHub where you store everything related to your project. It keeps your code, images, and documents together, making it easy to find what you need.

Why Create a Repository?

Track Changes: Keep an eye on all updates made to your project, like having an undo feature.

Teamwork Collaboration: Share your code with others, allowing everyone to see additions and edits.

Safe Storage: Store your code online for backup, so you won’t lose anything if your computer crashes.

Easy Access: With a repository, all your project files are in one place, so you can quickly locate and use them while you work.

Show Off Your Projects: Share your work with others by providing a link, showcasing your skills to potential employers. It serves as a portfolio to demonstrate your skills.

Integration with VS Code: Repositories work well with Visual Studio Code (a popular code editor that makes it easy to write and manage code). This integration allows you to sync changes directly to GitHub. Once you finish updating your code in VS Code, you can quickly push those changes to GitHub, keeping your projects organized and up-to-date online.

Setup

3. How to Install Git:

  • Go to git-scm.com and download Git.
  • Follow the install instructions for your computer.

4. How to Install Visual Studio Code (VS Code):

  • Visit code.visualstudio.com and download the right version for your device.
  • Install it by following the website's steps.

5. Installing Git on macOS

  • Open Terminal: Go to View > Terminal.
  • Install Xcode Command Line Tools:
    • Run this command in Terminal:
    • xcode-select --install
  • Follow the Prompts: A window will pop up asking if you want to install. Click “Install” to download Git.
  • Set VS Code as the Default Editor for Git:
    • After Git is installed, open VS Code.
    • Go to the terminal (View > Terminal) and run:
    • git config --global core.editor "code --wait"
    • This tells Git to use VS Code for editing.

6. Configuring Git in VS Code

Open VS Code.

Set Up Git Configurations:

  • Find Terminal by going to View > Terminal.

Run these commands:

  • git config --global user.name "Your Name"
  • git config --global user.email "your-email@example.com"
Verifying Your Configuration:

Check your settings by running:

  • git config --list

Make sure your username and email are correct.

7. Cloning a GitHub Repository

  • Open Visual Studio Code:
  • Open the Command Palette: Go to View > Command Palette or press Cmd + Shift + P.
  • Select Git: Clone: Type and select Git: Clone.
  • Paste the Repository URL: Paste the URL of the repository you want to clone (you can find this on GitHub).
  • Select a Folder: Choose a folder on your computer to save it and click Select Folder.

After Cloning: VS Code will create a new folder with the project files. You can open this folder to start working on your project!

8. Add Files and Push Project to GitHub

  • Open Your Project: Ensure your cloned repository is open in VS Code.
  • Add Your Project Files:
    • Copy your HTML and CSS files into the cloned folder. Having these files open in VS Code ensures they are ready to be pushed to GitHub.
  • Initialize Git (if needed):
    • If your repository is already cloned, you don’t need to run git init again—it’s already a Git repository.
    • If you do need to set it up, open the terminal in VS Code (go to Terminal > New Terminal) and type:
    • git init
  • Commit Your Changes:
    • In the terminal, type:
    • git add .

    The . tells Git to track all files (or add files) in the current directory, ensuring it remembers any changes you make.

    • Then type:
    • git commit -m "Initial commit"

    This command saves your changes in Git and adds a message ("Initial commit") to describe what you’ve done.

  • Push the Project to GitHub:
    • To upload your files to GitHub, type:
    • git push origin main

    (If you’re using a branch name like "master," replace main with your branch name.)

Branching with VS Code

  • What are Branches?

    Branches let you work on different features or fix bugs without messing up the main project. Use git status to see which branch you’re on.

  • Creating a Branch:

    To create a new branch (but not switch to it), type:

    git branch [branch-name]

  • Switching Branches:

    To switch to a branch and create it if it doesn’t exist, type:

    git switch -c [branch-name]

  • Merging Branches:

    To combine changes from one branch into your current branch, type:

    git merge [branch]

    This is often done during pull requests.

  • Deleting a Branch:

    To remove a branch that has been merged, type:

    git branch -d [branch-name]

Tips: Use clear names for branches, and make sure they focus on one task to keep things organized.

Working with Remote Repositories

  • Adding a Remote Repository:
    • Open your project in VS Code.
    • Open the terminal (go to View > Terminal).
    • To add a remote repository, type:

    git remote add origin <repository-URL>

    Replace <repository-URL> with the URL of your GitHub repository.

  • Pushing Changes:

    After you make changes and commit them, push them to the remote repository by typing:

    git push origin main

    (Replace main with your branch name if it’s different.)

  • Pulling Changes:

    To update your local repository with the latest changes from the remote, type:

    git pull origin main

    (Again, replace main with your branch name if needed.)

Pull Requests and Code Reviews

  • Creating a Pull Request:

    Once you've pushed your changes, navigate to your GitHub repository in your web browser.

    • Click the button to create a pull request for your new branch.
    • Add a concise title and a brief description, then click "Create Pull Request."
  • Reviewing Pull Requests:

    Access the "Pull Requests" tab in your GitHub repository.

    • Select the pull request you wish to review.
    • Examine the changes, leave comments, and engage with your team.
    • If everything looks good, approve the request or suggest changes as necessary.
